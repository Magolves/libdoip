Detailed Description

DoIP Server class to handle incoming DoIP connections and UDP messages.

This class manages the low-level TCP/UDP socket handling and provides a clean callback-based interface for application logic. The server can operate in two modes:

  1. Manual mode: User calls waitForTcpConnection() and manages the receive loop
  2. Automatic mode: User calls start() with a callback, server manages everything

Start the DoIP server with automatic connection handling.

This method sets up TCP and UDP sockets, spawns background threads to handle UDP messages and TCP connections, and returns immediately. The server continues running until stop() is called.

Parameters
onConnectionAcceptedCallback invoked when a new client connects. Return a DoIPServerModel to accept the connection, or std::nullopt to reject it.
sendAnnouncementsIf true, send vehicle announcements on startup
Returns
true if server started successfully, false otherwise
